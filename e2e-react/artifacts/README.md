# e2e-react visual evidence (before / after)

This directory stores the committed before/after screenshots and screen recordings that prove the visual fidelity of the two migrated screens (Kanban, Backlog) as AngularJS is replaced by React in-place.

## Git-tracked - do NOT gitignore

- Everything under `e2e-react/artifacts/` is committed evidence and MUST remain git-tracked.
- The repo-root `taiga-front/.gitignore` deliberately does NOT list `e2e-react/`; do not add it.
- Do NOT copy the legacy `e2e/screenshots/<browser>/.gitignore` trick (a `*` line plus `!.gitignore`), which ignores a directory's contents -- the exact opposite of what is needed here.
- The empty `baseline/` and `react/` directories are held by plain `.gitkeep` placeholder files (git cannot track empty directories); the placeholders are joined by the real captures once those are generated and committed.

## Layout

```
artifacts/
|-- baseline/   AngularJS captures (taken FIRST, before the AngularJS Kanban/Backlog code is removed)
`-- react/      React captures (taken AFTER the migration)
```

Note: when `npm run e2e` runs, Playwright also writes runtime folders `artifacts/<phase>/output/` (videos/traces) and an HTML report to `artifacts/report/`. These are generated by the run and are not the curated, committed screenshots.

## Two-phase capture workflow (order matters)

The order is an irreversible correctness constraint: the AngularJS screens can only be captured while they still exist.

1. Ensure the parent repo's `taiga-front` submodule pointer is at main HEAD `8a73e14` or later; otherwise local source changes are silently ignored by the deployed stack.
2. Seed sample data exactly once: `./taiga-manage.sh sample_data` -- run BEFORE the baseline pass and never again.
3. Baseline pass -- with the AngularJS screens still present, run the baseline capture and commit the results under `artifacts/baseline/`. This MUST happen before any Kanban/Backlog AngularJS code is removed.
4. Remove the AngularJS Kanban/Backlog screens and build the React bundle.
5. React pass -- run the capture against the React screens on the same, unchanged PostgreSQL volume (never reseeded or recreated between passes) and commit the results under `artifacts/react/`.
6. Compose the side-by-side comparison afterward from the two committed sets; no single test renders both frameworks at once.

## How captures are generated

- Generated by the isolated Playwright layer via `npm run e2e` (never `npm test`, which is the browserless Jest layer).
- Requires Node v16.19.1 and the deployed Docker/nginx stack on host port 9000.
- The capture phase is selected by the `CAPTURE_PHASE` environment variable: captures route into `react/` only when `CAPTURE_PHASE=react`, otherwise into `baseline/` (the default). This is configured in `../playwright.config.ts` (`outputDir: artifacts/${CAPTURE_PHASE}/output`) and used identically by the `../fixtures/` screenshot helper (`artifacts/<phase>/<section>/<name>.png`).

## Credentials & data consistency

- Every login -- at `createsuperuser` time and at every test login in both phases -- reads the same password: `TAIGA_ADMIN_PASSWORD` if set, otherwise the documented dev default `admin123`. The value must be identical by construction across create and login.
- The database is seeded once and the same volume is preserved unchanged across both capture passes.
